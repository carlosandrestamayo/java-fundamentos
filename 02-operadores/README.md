# Unidad 2 – Operadores y Expresiones en Java

## 🎯 Objetivo de la unidad

Comprender y utilizar correctamente los **operadores en Java** para construir
**expresiones** que permitan realizar cálculos, comparaciones y evaluaciones lógicas,
sentando las bases del comportamiento de los programas antes de introducir
estructuras de control.

Esta unidad se centra en **pensar las expresiones**, no en el flujo del programa.

---

## 📚 Contenidos

En esta unidad se abordan los siguientes temas:

- Operadores aritméticos (`+`, `-`, `*`, `/`, `%`)
- Operadores de asignación (`=`, `+=`, `-=`, `*=`, `/=`, `%=`)
- Operadores relacionales (`==`, `!=`, `>`, `<`, `>=`, `<=`)
- Operadores lógicos (`&&`, `||`, `!`)
- Operadores unarios (preincremento y postincremento)
- Operador ternario (`condición ? valor1 : valor2`)
- Precedencia y asociatividad de operadores
- Cortocircuito lógico (`&&` vs `&`, `||` vs `|`)
- Construcción de expresiones complejas

### Cómo usar este repositorio

1. Clona o descarga el repositorio en tu computadora.
2. Abre un IDE compatible con Java (IntelliJ IDEA, Eclipse, VS Code + Java).
3. Revisa primero teoria.md para comprender los conceptos.
4. Explora los ejemplos en ejemplos/ para ver implementaciones reales y prácticas.
5. Realiza los ejercicios en propuestos.md, escribiendo tu propio código.
6. Verifica tus respuestas comparándolas con soluciones.md.
7. Experimenta cambiando los valores de las variables para reforzar el aprendizaje.

---

## 🧠 Enfoque pedagógico

- No se utilizan estructuras de control (`if`, `for`, `while`) salvo que se indique.
- Se prioriza el razonamiento lógico antes de ejecutar el código.
- Se analizan los resultados paso a paso.
- Se refuerza la idea de que: *_Un programa es una secuencia de expresiones correctamente construidas._*

---

## ✅ Resultados de aprendizaje esperados

Al finalizar esta unidad, el estudiante será capaz de:

- Construir expresiones aritméticas y lógicas correctas.
- Diferenciar entre división entera y real.
- Predecir el resultado de una expresión sin ejecutar el programa.
- Utilizar el operador ternario como alternativa a decisiones simples.
- Evitar errores comunes relacionados con precedencia y cortocircuito.

---

## 💡 Idea clave de la unidad

- Los operadores no son solo símbolos, son las reglas que definen cómo piensa el programa.
- Dominar los operadores permite escribir código correcto, predecible y eficiente.
